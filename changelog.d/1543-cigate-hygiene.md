- **`CI Gate` aggregator no longer exits green prematurely, and no longer flags fork PRs awaiting approval as failed ([#1543](https://github.com/sceneview/sceneview/issues/1543)).** The gate had no lower bound on the checks it observed, so it could pass a code PR while a slow, non-path-gated workflow had not yet registered its check run. It now derives whether `ci.yml` is expected to run by replaying that workflow's `paths-ignore` against the PR's changed files, and — when expected — blocks the green exit until `ci.yml`'s three always-on core checks (`Detect changed paths`, `Repo hygiene checks`, `Quality gate (full)`) have registered; doc-only PRs that legitimately skip `ci.yml` are unaffected. Separately, the `action_required` conclusion — produced when a fork PR's workflow is awaiting maintainer approval — is no longer treated as a failure, so a fork PR pending review no longer shows a red `CI Gate`.
