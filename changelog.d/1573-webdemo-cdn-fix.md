- **Web demo: self-hosted models ([#1573](https://github.com/sceneview/sceneview/issues/1573)).** The `samples/web-demo` model catalog loaded every GLB — including the initial scene model — from `cdn.jsdelivr.net/gh/sceneview/sceneview@main/assets/models/glb/`. jsDelivr's gh-proxy returns HTTP 403 for those large GLB blobs, so `initSceneView()` never resolved and the demo stayed stuck on its loading overlay — the deployed viewer at `sceneview.github.io/web-demo/` was effectively broken. The 15-model curated catalog is now bundled under `samples/web-demo/src/jsMain/resources/models/` and loaded from a relative `models/` path, which works identically for local `jsBrowserDistribution` runs, the deployed site, and the Playwright device-QA suite (no external CDN). This matches SceneView's policy of always self-hosting demo assets. The Playwright `isIgnorableNoise` filter was also tightened so a genuine local model 403/404 now fails the suite instead of being silenced as "CDN noise".
